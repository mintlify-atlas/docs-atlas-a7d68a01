## Prerequisites

Before you begin, ensure your system meets these requirements:

<Note>
**Linux system with Wayland compositor** - Dice only works on Linux systems running a Wayland compositor (not X11). Common Wayland compositors include GNOME Shell (Wayland session), KDE Plasma (Wayland session), Sway, and Weston.
</Note>

- **C compiler** - GCC or Clang with C standard library support
- **Wayland development files** - While Dice doesn't link against libwayland, you still need the protocol headers for reference
- **Working Wayland session** - Verify with `echo $WAYLAND_DISPLAY` (should output something like `wayland-0`)

## Compilation

<Steps>
  <Step title="Navigate to the source directory">
    Open a terminal and navigate to where you've cloned or downloaded the Dice source code:
    
    ```bash
    cd ~/workspace/source
    ```
  </Step>

  <Step title="Compile the program">
    Compile `dice.c` using GCC with the following command:
    
    ```bash
    gcc -o dice dice.c -D_GNU_SOURCE
    ```
    
    This creates an executable named `dice` in the current directory.
    
    <Info>
    The `-D_GNU_SOURCE` flag enables GNU extensions needed for `memfd_create()` and other Linux-specific features.
    </Info>
  </Step>

  <Step title="Verify compilation">
    Check that the executable was created successfully:
    
    ```bash
    ls -lh dice
    ```
    
    You should see the `dice` executable file listed.
  </Step>
</Steps>

## Running the program

<Steps>
  <Step title="Execute Dice">
    Run the compiled program:
    
    ```bash
    ./dice
    ```
  </Step>

  <Step title="Observe the output">
    You should see diagnostic output in the terminal showing the connection process:
    
    ```
    INFO: xdg_runtime_dir_path: /run/user/1000
    INFO: display_name: wayland-0
    INFO: path: /run/user/1000/wayland-0
    SUCCESS: Successfully connected to sockfd
    ```
    
    The program will discover Wayland global interfaces and bind to them:
    
    ```
    Global Event:
     - Name: 1
     - Interface: wl_compositor
     - Version: 6
    Bind request for wl_compositor (ID: 3)
    ```
  </Step>

  <Step title="See the window">
    A 736x736 pixel window should appear on your screen displaying an image. The window will remain open, and the program will continue running in the terminal, printing protocol message details.
  </Step>

  <Step title="Exit the program">
    Press `Ctrl+C` in the terminal to terminate the program and close the window.
  </Step>
</Steps>

## What to expect

When Dice runs successfully, you'll observe:

1. **Terminal output** - Verbose logging showing each step of the Wayland protocol handshake, including object IDs, opcodes, and message sizes
2. **Window appearance** - A standalone window managed by your compositor, just like any other application window
3. **Event handling** - The program continues running to handle compositor events like configure requests and ping messages

## Troubleshooting

<Warning>
**"ERROR: XDG_RUNTIME_DIR is not set"** - You're not running in a proper user session. Make sure you're logged into a graphical session, not just a TTY console.
</Warning>

<Warning>
**"ERROR: Failed connection to socket"** - The Wayland compositor isn't running. Verify you're in a Wayland session (not X11) by running `echo $XDG_SESSION_TYPE`. It should output `wayland`.
</Warning>

### Common issues

#### Running on X11 instead of Wayland
If you're using GNOME or KDE, make sure you selected the Wayland session at login (not the default/X11 session). Look for "GNOME on Wayland" or "Plasma (Wayland)" in your login screen's session menu.

#### Permission denied errors
Ensure the socket at `$XDG_RUNTIME_DIR/$WAYLAND_DISPLAY` has proper permissions:

```bash
ls -l $XDG_RUNTIME_DIR/$WAYLAND_DISPLAY
```

It should be owned by your user account.

#### Compilation errors
If you see errors about missing headers or functions:

```bash
# Install build essentials (Debian/Ubuntu)
sudo apt-get install build-essential

# Install build essentials (Fedora)
sudo dnf install gcc glibc-devel

# Install build essentials (Arch)
sudo pacman -S base-devel
```

## Next steps

Now that you have Dice running, dive into the source code at `~/workspace/source/dice.c` to understand how each part works. Pay special attention to:

- `open_display_connection()` - How the Unix socket connection is established (dice.c:70)
- `bind_request()` - How protocol messages are formatted with proper padding (dice.c:150)
- `create_shm_pool()` - How file descriptors are passed using `sendmsg()` and `SCM_RIGHTS` (dice.c:295)
- The main event loop - How the program reads and responds to compositor messages (dice.c:516)
