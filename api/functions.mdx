## Display Connection

### get_display_path()

**Location**: dice.c:43

```c
static char *get_display_path(void)
```

Constructs the full path to the Wayland display socket by combining the XDG runtime directory with the display name.

#### Returns

Pointer to the allocated display path string (e.g., "/run/user/1000/wayland-0"), or NULL on error. The caller is responsible for freeing this memory.

#### Behavior

1. Reads `XDG_RUNTIME_DIR` environment variable
2. Reads `WAYLAND_DISPLAY` environment variable (defaults to "wayland-0" if not set)
3. Allocates and returns the combined path
4. Logs diagnostic information to stderr

---

### open_display_connection()

**Location**: dice.c:70

```c
static int open_display_connection(void)
```

Establishes a Unix domain socket connection to the Wayland display server.

#### Returns

Socket file descriptor on success, or -1 on error.

#### Behavior

1. Gets the display path using `get_display_path()`
2. Creates a `AF_UNIX` socket with `SOCK_STREAM` type
3. Connects to the socket at the display path
4. Returns the connected socket file descriptor

### read_msg()

**Location**: dice.c:120

```c
static int read_msg(int sockfd, struct Request *req)
```

Reads a complete Wayland protocol message from the socket.

#### Parameters

<ParamField path="sockfd" type="int" required>
  Socket file descriptor for the Wayland connection
</ParamField>

<ParamField path="req" type="struct Request*" required>
  Pointer to request structure to populate with received data. The caller must free req->data if non-NULL.
</ParamField>

#### Returns

0 on success, -1 on error.

#### Behavior

1. Reads the message header from the socket
2. If the message has payload data (size > header size), allocates memory and reads it
3. Sets req->data to NULL and req->data_len to 0 if no payload
4. Returns error on read failure or unexpected EOF

## Protocol Operations

### bind_request()

**Location**: dice.c:150

```c
static void bind_request(int sockfd, uint32_t registry_id, uint32_t name,
                         const char *interface, uint32_t version,
                         uint32_t new_id)
```

Sends a `wl_registry::bind` request to bind a global interface to a new object ID.

#### Parameters

<ParamField path="sockfd" type="int" required>
  Socket file descriptor for the Wayland connection
</ParamField>

<ParamField path="registry_id" type="uint32_t" required>
  Object ID of the wl_registry object
</ParamField>

<ParamField path="name" type="uint32_t" required>
  The name of the global object as advertised by the registry
</ParamField>

<ParamField path="interface" type="const char*" required>
  Interface name string (e.g., "wl_compositor", "wl_shm")
</ParamField>

<ParamField path="version" type="uint32_t" required>
  Protocol version number to bind
</ParamField>

<ParamField path="new_id" type="uint32_t" required>
  New object ID to assign to the bound interface
</ParamField>

#### Behavior

1. Calculates the required payload size including string padding
2. Constructs the bind request with proper alignment
3. Sends the request via `write_msg()`
4. Exits on failure

</ParamField>
